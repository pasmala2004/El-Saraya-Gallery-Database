2. Database Layer (11 Tables)
   Core Entities:
   - customers: Client management with location tracking
   - product_categories: Product classification
   - products: Sellable product catalog
   
   Quotation System:
   - quotations: Price estimates with workflow (draft → sent → approved)
   - quotation_items: Line items with pricing
   
   Job Management:
   - jobs: Production lifecycle tracking (7 lifecycle dates)
   - measurements: Site visit records with measurer tracking
   - measurement_items: Detailed dimensions per piece (width/height)
   - payments: Milestone-based payments (Deposit/Production/Final)
   - activity_logs: Complete audit trail for jobs
   
   Reporting:
   - reports: Generated report metadata and file paths

3. Database Schema Features
   ✅ UUID primary keys (gen_random_uuid)
   ✅ Timezone-aware timestamps (created_at, updated_at)
   ✅ 5 PostgreSQL enum types for type safety
   ✅ 13 foreign key constraints (8 CASCADE + 5 RESTRICT)
   ✅ 14 indexes for query optimization
   ✅ 3 unique constraints for data integrity
   ✅ Server-side defaults for all critical fields
   ✅ Two-layer model hierarchy (Base → BaseEntity → Models)

4. Business Domain Enums

🚀 Getting Started
==================

1. Start database:
   docker compose up -d db

2. Run migrations:
   alembic upgrade head

3. Verify:
   docker compose exec -T db psql -U erp_user -d erp_db < docs/verify_migration.sql

4. Start application:
   uvicorn app.main:app --reload

5. Test:
   curl http://localhost:8000/api/v1/health
