## Technical Details

### Idempotency Implementation
Each seeder checks for existing records before creating new ones:

```python
result = await session.execute(
    select(Model).where(Model.unique_field == value)
)
existing = result.scalar_one_or_none()

if existing:
    return existing
else:
    record = Model(...)
    session.add(record)
    return record
```

### Dependency Order
1. Product Categories (no dependencies)
2. Products (depends on categories)
3. Customers (no dependencies)
4. Quotations & Items (depends on customers and products)
5. Jobs (depends on approved quotations)
6. Payments (depends on jobs)
7. Measurements & Items (depends on jobs)
8. Activity Logs (depends on jobs)

### Payment Split Logic
For each job, 3 payments are created:
- Deposit: 70% of quotation final price
- Production: 20% of quotation final price
- Final: 10% of quotation final price

Status and dates vary realistically based on job status.
